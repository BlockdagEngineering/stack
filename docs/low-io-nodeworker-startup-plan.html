Low-IO Nodeworker Startup

This release-stack change removes a recurring startup cost: recursively chowning large chain data volumes on every node container boot. The new behavior fixes ownership once per mounted path and then uses a marker to avoid repeating expensive drive IO.

Repo: pool-stack-docker Branch: feature/release-low-io-nodeworker-startup Risk: release packaging Live action: none

Changes

Ownership Sentinel

For each persisted path, the entrypoint checks a marker and root path owner. If both are valid, it skips recursive ownership repair.

Operator Override

BDAG_FIX_VOLUME_OWNERSHIP=auto|always|never controls whether ownership repair is automatic, forced, or skipped.

Future Health Flags

NODEWORKER_EXTRA_ARGS lets releases enable new nodeworker flags, including mining-readiness gates, without rebuilding the image.

Acceptance Criteria

  • Default behavior remains safe for first-time users: ownership is repaired on first boot.
  • Later boots skip recursive chown when markers show paths were already fixed.
  • Operators can force or disable ownership repair through one environment variable.
  • Nodeworker extra arguments are disabled by default and documented in .env.example.
  • Only static validation is run here; no Docker services are restarted.
