FastSnap Maintenance Handoff

This release feature makes public snapshot.bdsnap seeding safe and default on a dual-node mining host. The pool must first drain the node that will be stopped, prove that ASIC work is routed to the other node, and only then allow snapshot export.

Primary Risk Removed
Pool mines into stopped node

The old script guessed from metrics. The new path asks the pool router to drain a backend and refuses to proceed if the router cannot switch away.

Safety Contract
Fail Closed

If no healthy alternate exists, the maintenance API returns conflict and no node is stopped.

Release Default
Hot artifact sidecar

ops/install-p2p-services.sh installs bdag-fastartifact-sidecar.timer unless disabled with BDAG_FASTSNAP_SEED_TIMER_ENABLED=0.

Handoff Flow

1. InspectRead pool_rpc_backend_selected from the pool metrics endpoint.
2. DrainCall /admin/rpc-backend-maintenance for the non-active backend.
3. ProveRequire selected backend unchanged and pool_rpc_backend_maintenance=1 for the export backend.
4. ExportStop only the drained node and run bdag snap export against its datadir.
5. RestoreRestart the node, clear maintenance, verify archive, and hardlink it into node datadirs.

Required Pool Source

ComponentRequirementReason
Pool router/admin/rpc-backend-maintenanceProvides an explicit drain contract instead of shell-script inference.
Metricspool_rpc_backend_maintenance and pool_rpc_backend_maintenance_until_secondsLets automation and dashboards prove a backend is intentionally unavailable.
Health JSONBackend maintenance fields under /health/job-stateLets operators distinguish planned snapshot work from node failure.

Configuration

POOL_RUNTIME_ADMIN_ENABLED=true
POOL_RPC_ROUTER_ENABLED=true
POOL_RPC_BACKENDS=node1=http://bdag-miner-node-1:38131,node2=http://bdag-miner-node-2:38131
POOL_RPC_ROUTER_TEMPLATE_LANE_MODE=active-passive

BDAG_POOL_ADMIN_URL=http://127.0.0.1:9092
BDAG_FASTSNAP_NODE1_SERVICE=bdag-miner-node-1
BDAG_FASTSNAP_NODE2_SERVICE=bdag-miner-node-2
BDAG_FASTSNAP_NODE1_DATADIR=/opt/blockdag-pool/data/node1
BDAG_FASTSNAP_NODE2_DATADIR=/opt/blockdag-pool/data/node2

Operator Command

cd /path/to/pool-stack
./ops/install-p2p-services.sh
systemctl --user status bdag-fastartifact-sidecar.timer
systemctl --user start bdag-fastartifact-sidecar.service
