BlockDAG P2P Interface Discovery Standard

Pool releases should treat LAN and VPN interfaces as first-class P2P paths. The standard is interface-agnostic firewall allowance plus deterministic local peer lists for two-node deployments.

Firewall

All configured interfaces eligible

bdag-p2p-firewall.service accepts configured BlockDAG P2P ports in host INPUT and Docker DOCKER-USER without an interface filter.

Local nodes

Cross-peer by default

update-local-peers.py writes node-specific addpeer lists so node1 and node2 can dial each other through Docker DNS and host LAN/VPN addresses.

Operator peers

LAN/VPN pinning supported

Known peers can be supplied through LAN_PEER_ADDRESSES, VPN_PEER_ADDRESSES, ZEROTIER_PEER_ADDRESSES, or EXTRA_PEER_ADDRESSES.

Firewall Contract

The P2P firewall helper is intentionally broad over interfaces and narrow over ports. This lets a host add or remove VPN interfaces without editing firewall unit files, while still limiting the rule to configured P2P ports.

BDAG_P2P_PORTS=8151,8152
BDAG_P2P_PROTOCOLS=tcp
systemctl enable --now bdag-p2p-firewall.service

If a future node build enables UDP-based discovery, set BDAG_P2P_PROTOCOLS=tcp,udp in /etc/default/bdag-p2p-firewall and publish matching Docker UDP ports.

Peer List Contract

SourcePurpose

Host LAN IPv4 addressesAllow peers on the same physical LAN to connect when reachable.

Operational Boundary

update-local-peers.py --apply recreates node containers only when the active peer env changes. Run it during install or an approved maintenance action. A plain update-local-peers.py run is safe for verification because it updates env only and does not recreate containers.
