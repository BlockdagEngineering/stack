BlockDAG IPFS Content Sidecar

The IPFS content sidecar makes finalized BlockDAG FastArtifact generations easier to find and fetch. The raw datadir sidecar now seals immutable file/chunk generations with signed manifest roots; IPFS still must never decide chain trust from a CID alone.

Design Rules

  • ops/seal_rawdatadir_sidecar_content.py turns each completed sidecar rsync into immutable SHA-256 chunks plus a signed canonical manifest root.
  • ops/ipfs_content_sidecar.py pins only an already-finalized signed artifact directory without a DO_NOT_PUBLISH marker.
  • Raw datadir artifact generation remains owned by ops/publish-rawdatadir-artifact.sh and is retried separately.
  • Final publishability remains gated by artifact finalization, sidecar content sealing, manifest signatures, and consensus validation.
  • Background work runs only after background_maintenance_decision() allows it; pressure defers the tick rather than disabling future attempts.
  • FastArtifact source eligibility is recorded with each attempt, but it is not a permanent gate that blocks pinning an already-finalized safe artifact.
  • DO_NOT_PUBLISH markers and unsigned manifests fail closed by default.
  • The generated latest index records the artifact CID and manifest hash, but receivers must still verify the signed manifest root, chunk/file hashes, network, genesis, tip, state root, and then catch up through normal consensus.

Modes

  • BDAG_IPFS_CONTENT_SIDECAR_MODE=off: disabled.
  • BDAG_IPFS_CONTENT_SIDECAR_MODE=auto: timer wakes lazily, retries after pressure clears, and pins only when all gates pass and a finalized safe artifact exists.
  • BDAG_RAWDATADIR_SIDECAR_CONTENT_MODE=auto: rawdatadir sidecar syncs also seal immutable file/chunk generations.

Release Safety

Installer logic keeps retry timers active in auto mode. The services self-defer when mining pressure, unsafe storage, missing finalization, unsigned manifests, or DO_NOT_PUBLISH markers make work unsafe. Single active-node mining systems remain fail-closed for publication unless a finalized artifact already exists and resource pressure has cleared.

Status

The sidecar writes ops/runtime/ipfs-content-sidecar-status.json and ops/runtime/ipfs-content/latest-index.json. Dashboard/watchdog code can consume those files without probing IPFS on every refresh.
