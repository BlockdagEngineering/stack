Codex Intelligent Mining Supervisor

This feature adds a user-level systemd timer that periodically invokes codex exec with a compact mining health packet. It exists because scripted watchdogs can repair known faults, but they cannot reason about new faults, patch source, update the backlog, and create durable evidence.

What It Does

Every 15 minutes after the previous run finishes, it snapshots dashboard health, pool metrics, systemd guard state, and recent incidents, then asks Codex to diagnose and act according to the mining runbooks.

What It Avoids

The wrapper itself does not restart pools, nodes, Docker, or ASICs. It delegates decisions to Codex with explicit production-safety instructions and the current health context.

Why It Matters

Mining failures cost money by the minute. This preserves deterministic guards for fast known repairs while adding intelligent analysis for novel degradation, code fixes, and backlog creation.

Design Notes

OnUnitInactiveSec=15min is used instead of a fixed wall-clock interval so a slow Codex analysis cannot overlap itself. A file lock is also held during each run. This keeps token burn intentional and prevents multiple supervisor agents from racing each other.
