# Dashboard Map

Grafana dashboards are generated by `scripts/generate_dashboards.py` and provisioned from `grafana/dashboards`. They use Prometheus and Loki datasource UIDs from `grafana/provisioning/datasources/datasources.yml`.

All dashboards include quick ranges for 1h, 6h, 12h, 24h, 3d, 7d, and 30d.

## Provisioned Dashboards

| Dashboard | File | Purpose |
| --- | --- | --- |
| BDAG Operator Cockpit | `grafana/dashboards/operator-cockpit.json` | Default home view for pool operators: revenue velocity, per-miner income, accepted work share, miner freshness, dual-node health, pool template cadence, block-submit quality, global pool position, and recent incidents. |
| BDAG Observed Peer IPs | `grafana/dashboards/observed-peers.json` | Public P2P peer IP count, best-guess peer location, country/ASN/city distribution, observed peer detail, and peer-related node logs. |
| BDAG Overview | `grafana/dashboards/overview.json` | Stack status, connected miners, share age, sync progress, pool activity, work share, node heights. |
| BDAG Miners | `grafana/dashboards/miners.json` | Managed/OK/connected miner counts, work percentage by miner, share age, hashrate, hardware errors. |
| BDAG Pool | `grafana/dashboards/pool.json` | Share stall, job stall, template-frozen state, share/submit recent windows, pool ages, block submit outcomes. |
| BDAG Nodes | `grafana/dashboards/nodes.json` | Node lag, import freshness, old dashboard node state, native node heights, tips, unsequenced blocks, template/P2P errors. |
| BDAG Earnings | `grafana/dashboards/earnings.json` | Wallet balance/rate, BDAG price, miner USD/h, miner BDAG/h, global pool share. |
| BDAG Host And Containers | `grafana/dashboards/system.json` | Host CPU, root disk free, PostgreSQL exporter health, memory, production container CPU. |
| BDAG Thermals | `grafana/dashboards/thermals.json` | Host thermal zones and CPU frequency from node exporter. |
| BDAG Logs And Incidents | `grafana/dashboards/incidents.json` | Pool/node errors, watchdog/repair events, API scrape duration. |

## Old Dashboard Parity

| Old Dashboard Area | New Target | Status |
| --- | --- | --- |
| Pool Status tab | Operator Cockpit, Overview, Pool, Nodes, System | Covered for read-only monitoring. Old controls remain old-control-only. |
| Miners tab | Operator Cockpit, Miners, Pool, Incidents | Covered for identity, status, work share, share age, hashrate, and configuration state. Old configure/restart actions remain old-control-only. |
| Global tab | Operator Cockpit, Earnings and Overview | Covered for global latest block, pool share, blocks, and estimated USD/h. |
| Earnings tab | Operator Cockpit, Earnings | Covered for wallet balance, BDAG/h, USD/h, price, and miner estimates. |
| Repair/watchdog timeline | Incidents | Covered through Loki once staged run starts. Existing repair automation stays in the old stack. |

## Browser Safety

The dashboards do not call the old dashboard APIs directly from browser JavaScript. Grafana reads Prometheus and Loki only. The only old-dashboard browser exposure is a link to `http://127.0.0.1:8088` for the existing repair console.

## Generation

Regenerate dashboards after changing `scripts/generate_dashboards.py`:

```bash
cd /home/jeremy/blockdag-asic-pool/ops/observability
python3 scripts/generate_dashboards.py
```

The validation script regenerates them automatically.
